EYES ON DUTY

Eyes on Duty is a mobile horror observation game made in Unity, inspired by the I’m on Observation Duty series. Players monitor security cameras in a haunted restaurant, spotting and reporting paranormal anomalies before they spiral out of control. I built the core systems — anomaly detection, camera switching, UI flow, and game management — all optimized for mobile performance. The game blends tense atmosphere, smooth controls, and immersive audio, showcasing my ability to design complete gameplay systems with strong UI/UX and platform polish.

Development Journey

The Idea

Eyes on Duty was my first full mobile project — a real test of turning a simple concept into a complete, polished experience. Inspired by I’m on Observation Duty, I wanted to create a tense observation-style horror game where players monitor several camera feeds inside a restaurant and report strange anomalies before they get out of control.

How I Planned It

The initial plan was straightforward: replicate the core gameplay loop — camera switching, spotting anomalies, and reporting them. But during development, I started focusing on the systems beneath the surface. I wanted the anomalies to feel unpredictable, so I built a structure that could support random spawns, different anomaly types, and a gradual difficulty curve. The goal became less about copying a game and more about understanding how to build a modular and replayable system around the same idea.

How I Build It

I developed the anomaly management system entirely in C#, centered around the AnomalyManager which controls what happens, where, and when. Each camera feed has its own anomaly configuration, and the manager ensures only available cameras can receive new anomalies. The AnomalyReportingSystem ties directly into that logic — letting the player select a camera, choose a report type, and receive real-time feedback.

  • Procedural anomaly timing ensures unpredictability.
  • Smooth UI transitions make camera switching feel natural and cinematic.
  • A modular structure allows new cameras and anomaly types to be added without rewriting existing code.
  • Optimized for mobile, with touch controls, audio balancing, and low memory usage across different screen sizes.

I also implemented a simple penalty system (three wrong reports = game over) and ambient sound design to build immersion without relying on heavy visuals.

What Needs Work

The current version works well as a proof of concept, but it still has room to grow. I want to expand the number of anomalies, refine the pacing to feel more “alive,” and introduce subtler visual cues to reward sharp-eyed players. With more content and polish, Eyes on Duty could become a fully realized mobile horror title and not just a prototype.
